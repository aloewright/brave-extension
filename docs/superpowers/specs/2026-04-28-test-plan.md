# AI Dev Sidebar — Manual Test Plan (M8 / ALO-253)

This document covers everything that **must be verified by hand on a real Brave install** with the native host running. Anything that can be exercised headlessly is in the Playwright suite under `tests/e2e/`.

## Setup once

1. Install dev dependencies (Playwright is now part of `package.json`):
   ```
   pnpm install
   pnpm exec playwright install chromium     # required before `pnpm test:e2e`
   ```
2. Build the extension:
   ```
   pnpm build
   ```
3. Load `build/chrome-mv3-prod` as an unpacked extension in Brave (`brave://extensions` → Developer mode → Load unpacked).
4. Run `pnpm install-host` (and `pnpm install-host --enable-terminal-path` once you reach §1.b).

## Running the automated suites

- Unit/integration: `pnpm test` (vitest).
- E2E (sidepanel rail, consent banner, references tray): `pnpm test:e2e`.
- Both must be green before sign-off.

## 1. Install + register (M7)

### 1.a Clean install
- [ ] On a clean machine (or after `pnpm uninstall-host`), run `pnpm install-host`.
- [ ] `~/.config/ai-dev-sidebar/mcp-token` exists and is mode `0600`.
- [ ] `~/.config/ai-dev-sidebar/env` exists and is mode `0600`.
- [ ] Native messaging manifest installed at the OS-correct path:
  - macOS: `~/Library/Application Support/BraveSoftware/Brave-Browser/NativeMessagingHosts/com.aloe.ai_dev_sidebar.json`
  - Linux: `~/.config/BraveSoftware/Brave-Browser/NativeMessagingHosts/com.aloe.ai_dev_sidebar.json`
- [ ] `~/.claude.json` has an `mcpServers."ai-dev-sidebar"` entry pointing at the local HTTP+SSE endpoint.
- [ ] Pre-existing siblings under `mcpServers` are preserved untouched.

### 1.b Terminal-path enable
- [ ] Run `pnpm install-host --enable-terminal-path`.
- [ ] `~/.zshrc` and `~/.bashrc` each contain a single, marked `# >>> ai-dev-sidebar >>>` … `# <<< ai-dev-sidebar <<<` block adding `~/.config/ai-dev-sidebar` to `PATH`.
- [ ] `~/.config/ai-dev-sidebar/claude` wrapper exists and is executable (`-rwx------`).
- [ ] In a fresh shell: `which claude` resolves to that wrapper, `claude --version` runs.

### 1.c Idempotent reinstall
- [ ] Run `pnpm install-host` again (no flag) — exits 0, no diff applied.
- [ ] Run `pnpm install-host --enable-terminal-path` again — no duplicate marker blocks in `.zshrc`/`.bashrc`.

### 1.d Uninstall
- [ ] Run `pnpm uninstall-host`.
- [ ] Native messaging manifest removed.
- [ ] `~/.claude.json` `mcpServers."ai-dev-sidebar"` removed; siblings preserved.
- [ ] Marker block removed from `.zshrc` / `.bashrc`.
- [ ] Wrapper `~/.config/ai-dev-sidebar/claude` removed.
- [ ] Token + env files removed.

## 5. Element picker + references (M4)

> Automated coverage: `tests/e2e/references-tray.spec.ts` validates that pre-seeded references render in the tray + count badge. The end-to-end picker flow needs a real tab and screenshot capture and must be verified manually below.

- [ ] Open any web page in the active tab.
- [ ] In the Terminal section, click `[+ Reference]`.
- [ ] Hover over elements on the page — outline overlay follows the cursor.
- [ ] Click an element. Picker exits, a chip appears in the references tray.
- [ ] Click the chip → preview popover shows selector, screenshot, outerHTML, textContent.
- [ ] Drag the chip into the terminal — `@ref_<ulid>` token is inserted at the cursor position.
- [ ] Click `×` on the chip — chip disappears; the corresponding MCP resource is also removed (from `claude` run `resources/list` and confirm it's gone).
- [ ] Run `claude` and ask it to read the reference content — it returns the captured outerHTML/textContent.

## 9. Recorder (M6)

- [ ] In the Recorder section, pick **Tab** as source. Click **Start**.
- [ ] Recording indicator appears; timer increments.
- [ ] Click **Stop**. Browser download triggers a file at `~/Downloads/recording-<ISO>.mp4`.
- [ ] Mirror copy exists at `~/.config/ai-dev-sidebar/recordings/<ulid>.mp4`.
- [ ] Recording metadata appears in the Recorder history list.
- [ ] In `claude`, call `recorder_list` — sees the new recording.
- [ ] Call `recorder_get({id})` — returns metadata + a `file://` URI to the mirror copy.
- [ ] Repeat for **Screen** source.
- [ ] Repeat for **Camera** source.
- [ ] Long recording (>2 min) doesn't OOM the service worker — offscreen document does the streaming, not the background page. Verify by checking Brave's task manager: the offscreen frame's memory grows, the SW stays flat.

> Automated coverage: `tests/e2e/consent-banner.spec.ts` covers the banner UI for both `write` and `always-prompt` classes plus the queueing behaviour. Below verifies the end-to-end FSM with the real background.

## 11. Edge cases

- [ ] **Native host crash**: kill the host process (`pkill -f ai-dev-sidebar-host` or platform equivalent). Sidepanel shows a "host disconnected" banner and auto-reconnects within a few seconds.
- [ ] **Port 8473 in use**: start something else on `:8473` first (e.g. `python3 -m http.server 8473`), then start the host. It falls back to `8474` (or next free port) and re-registers `~/.claude.json` with the new URL.
- [ ] **`~/.claude.json` siblings**: pre-populate `mcpServers."some-other"`. Run `pnpm install-host`. Confirm the install merges its own entry without overwriting the sibling.
